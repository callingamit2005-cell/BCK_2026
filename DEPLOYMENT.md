linting issues (`npm run lint`)\n- [ ] Code reviewed by at least 1 other developer\n- [ ] No `console.log()` statements left\n- [ ] No hardcoded secrets or API keys\n\n### Testing\n- [ ] Manually tested on local environment\n- [ ] Tested on mobile devices\n- [ ] Tested in different browsers (Chrome, Safari, Firefox)\n- [ ] Dark mode tested\n- [ ] Accessibility tested (keyboard navigation)\n- [ ] Network throttling tested (slow connections)\n\n### Documentation\n- [ ] Updated CHANGELOG.md\n- [ ] Updated API documentation if changes made\n- [ ] Updated README if new features\n- [ ] All code comments are current\n\n### Database\n- [ ] All migrations written\n- [ ] Migration tested locally\n- [ ] Backup procedure verified\n- [ ] RLS policies reviewed\n\n---\n\n## Frontend Deployment (Vercel)\n\n### Automatic Deployment (Recommended)\n\nVercel automatically deploys on push to main/production branch.\n\n#### Setup (One-time)\n\n1. **Connect Repository**:\n   - Go to https://vercel.com\n   - Click \"New Project\"\n   - Select your GitHub repository\n   - Click \"Import\"\n\n2. **Configure Project**:\n   ```\n   Framework: Vite\n   Root Directory: ./\n   Build Command: npm run build\n   Output Directory: dist\n   ```\n\n3. **Add Environment Variables**:\n   - Go to Project Settings → Environment Variables\n   - Add:\n     ```\n     VITE_SUPABASE_URL=https://...\n     VITE_SUPABASE_ANON_KEY=...\n     VITE_ENABLE_ANALYTICS=true\n     ```\n\n#### Deploy Process\n\n```bash\n# Local: Make changes and commit\ngit add .\ngit commit -m \"feat: new feature\"\ngit push origin feature-branch\n\n# GitHub: Create Pull Request\n# → Vercel generates preview URL\n\n# Review & Test\n# → Test on preview URL\n# → Get reviews\n\n# Merge to main\ngit checkout main\ngit merge feature-branch\ngit push origin main\n\n# Vercel: Auto-deploys to production\n# → Available at https://your-domain.com\n```\n\n### Manual Deployment CLI\n\nIf needed, deploy manually:\n\n```bash\n# Install Vercel CLI\nnpm i -g vercel\n\n# Login to Vercel\nvercel login\n\n# Deploy to production\nvercel --prod\n\n# Deploy to staging\nvercel\n```\n\n### Vercel Deployment Stages\n\n1. **Preview**: Every PR gets unique preview URL\n2. **Staging**: Optional staging environment\n3. **Production**: Main live site\n\n---\n\n## Backend Deployment (Supabase)\n\n### Migrations\n\n#### Create Migration\n\n```bash\n# Create new migration file\nsupabase migration new add_new_table\n\n# Edit file and write SQL\n# Location: supabase/migrations/20260227_add_new_table.sql\n```\n\n#### Deploy Migration\n\n```bash\n# Push migrations to production\nsupabase db push\n\n# Verify migration ran\n# Go to Supabase Studio → Migrations tab\n```\n\n### Edge Functions Deployment\n\n```bash\n# Edit edge function\n# Location: supabase/functions/my-function/index.ts\n\n# Deploy function\nsupabase functions deploy send-invite\n\n# Test function\nsupabase functions invoke send-invite \\\n  --body '{\"email\": \"test@example.com\"}'\n```\n\n### Supabase CLI Commands\n\n```bash\n# Check status\nsupabase status\n\n# View logs\nsupabase logs\n\n# Pull from production\nsupabase db pull\n\n# Reset local (careful!)\nsupabase db reset\n\n# Link to project\nsupabase link --project-ref project-id\n```\n\n---\n\n## Environment Variables\n\n### Vercel Environment Variables\n\nSet in Vercel Dashboard → Settings → Environment Variables\n\n```env\n# Production\nVITE_SUPABASE_URL=https://project.supabase.co\nVITE_SUPABASE_ANON_KEY=eyJhbGc...\nVITE_ENABLE_ANALYTICS=true\nVITE_API_URL=https://project.supabase.co\n\n# Staging\nVITE_SUPABASE_URL=https://staging-project.supabase.co\nVITE_ENABLE_ANALYTICS=false\n```\n\n### Local Development\n\nCreate `.env.local`:\n\n```env\nVITE_SUPABASE_URL=http://localhost:54321\nVITE_SUPABASE_ANON_KEY=eyJhbGc...\nVITE_ENABLE_ANALYTICS=false\n```\n\n### Security Rules\n\n- ✅ Store in Vercel/Supabase dashboards\n- ✅ Use different keys per environment\n- ❌ Never commit to Git\n- ❌ Never log secrets\n- ❌ Never hardcode values\n\n---\n\n## Database Migrations\n\n### Safe Migration Process\n\n1. **Test Locally**:\n   ```bash\n   supabase db reset  # Reset to clean state\n   supabase migration new test_migration\n   # Edit migration file\n   supabase db push  # Test locally\n   ```\n\n2. **Create Backup**:\n   ```\n   Supabase Studio → Backups → Download backup\n   ```\n\n3. **Deploy**:\n   ```bash\n   supabase db push  # Runs on production\n   ```\n\n4. **Verify**:\n   ```\n   Supabase Studio → SQL Editor → Check tables\n   ```\n\n### Handling Migration Errors\n\nIf migration fails:\n\n```bash\n# Check recent migrations\nsupabase migration list\n\n# View migration logs\nsupabase logs\n\n# Rollback (if supported)\nsupabase migration down\n\n# Or restore from backup\n# Supabase Studio → Backups → Restore\n```\n\n---\n\n## Monitoring & Health Checks\n\n### Vercel Monitoring\n\n1. **Analytics**: https://vercel.com/dashboard\n   - Response times\n   - Error rates\n   - Request patterns\n\n2. **Logs**:\n   ```\n   Vercel Dashboard → Project → Deployments → Logs\n   ```\n\n3. **Uptime**: https://upstatus.com or similar service\n\n### Supabase Monitoring\n\n1. **Database**:\n   ```\n   Supabase Studio → Reports\n   - Query performance\n   - Replication status\n   - Resource usage\n   ```\n\n2. **API Logs**:\n   ```\n   Supabase Studio → SQL Editor → Recent queries\n   ```\n\n3. **Edge Functions**:\n   ```\n   Supabase Studio → Functions → Logs\n   ```\n\n### Health Check Endpoint\n\nAdd to application:\n\n```typescript\n// src/pages/api/health.ts (or route handler)\napp.get('/api/health', (req, res) => {\n  res.json({\n    status: 'healthy',\n    timestamp: new Date().toISOString(),\n    uptime: process.uptime()\n  });\n});\n```\n\nMonitor with:\n\n```bash\n# Automated monitoring\ncurl https://your-app.com/api/health\n\n# Expected:\n# { \"status\": \"healthy\" }\n```\n\n---\n\n## Rollback Procedures\n\n### Frontend Rollback\n\n#### Option 1: Vercel Dashboard\n\n1. Go to Vercel Dashboard\n2. Select Project → Deployments\n3. Find previous successful deployment\n4. Click \"...\" → Promote to Production\n\n#### Option 2: Git Revert\n\n```bash\n# Find commit hash of previous good version\ngit log --oneline\n\n# Revert to that commit\ngit revert abc123\ngit push origin main\n\n# Vercel auto-deploys\n```\n\n### Backend Rollback\n\n#### Restore from Backup\n\n1. Supabase Studio → Backups\n2. Select point-in-time before issue\n3. Click \"Restore\"\n4. Confirm (this is destructive)\n\n#### Revert Migration\n\n```bash\n# If migration can be reversed\nsupabase migration down\n\n# Or create reverse migration\nsupabase migration new revert_bad_migration\n# Write reverting SQL\nsupabase db push\n```\n\n---\n\n## Performance Optimization\n\n### Vercel Performance\n\n```bash\n# Build optimization\nbun run build\n\n# Check bundle size\nls -lah dist/\n\n# Expected: < 500KB chunk size\n```\n\n### Database Performance\n\n```sql\n-- Check slow queries\nSELECT query, mean_time FROM pg_stat_statements\nORDER BY mean_time DESC LIMIT 10;\n\n-- Add indexes if needed\nCREATE INDEX idx_expenses_user_created \nON expenses(user_id, created_at DESC);\n```\n\n### Caching Strategy\n\n```tsx\n// React Query caching\nconst { data } = useQuery(\n  ['expenses'],\n  fetchExpenses,\n  { staleTime: 5 * 60 * 1000 } // 5 min cache\n);\n```\n\n### Image Optimization\n\n```tsx\n// Use Next.js-style optimization\nconst imgSrc = new URL('image.png', import.meta.url).href;\n<img src={imgSrc} alt=\"description\" />\n```\n\n---\n\n## Post-Deployment Verification\n\n### Smoke Tests\n\nAfter deployment, test:\n\n```\n[ ] App loads without errors\n[ ] Login/signup works\n[ ] Add expense works\n[ ] Groups feature works\n[ ] Analytics loads\n[ ] Mobile responsive\n[ ] Dark mode toggle works\n[ ] API calls succeed\n[ ] Database queries work\n```\n\n### Performance Check\n\n```bash\n# Check Core Web Vitals\n# Use: https://pagespeed.web.dev/\n\n# Should be Green:\n# - Largest Contentful Paint (LCP): < 2.5s\n# - First Input Delay (FID): < 100ms\n# - Cumulative Layout Shift (CLS): < 0.1\n```\n\n### Error Monitoring\n\n```\nCheck Vercel Logs for:\n- No 500 errors\n- No unhandled exceptions\n- No database connection errors\n```\n\n---\n\n## Security Checklist\n\n### Pre-Deployment\n\n- [ ] No secrets in code\n- [ ] No hardcoded API keys\n- [ ] Environment variables set\n- [ ] HTTPS enabled (auto with Vercel)\n- [ ] CORS configured\n- [ ] Rate limiting enabled\n- [ ] Input validation implemented\n- [ ] Authentication required for protected routes\n\n### Post-Deployment\n\n- [ ] Test with wrong credentials (should fail)\n- [ ] Test with tampered tokens (should fail)\n- [ ] Test CORS (should reject invalid origins)\n- [ ] Test rate limiting (should throttle)\n- [ ] Verify TLS/SSL certificate\n- [ ] Check security headers\n\n### Ongoing\n\n```bash\n# Monitor dependencies for vulnerabilities\nbun audit\nnpm audit\n\n# Update regularly\nbun install --latest\n```\n\n---\n\n## Troubleshooting Deployments\n\n### Build Fails\n\n```bash\n# Check local build first\nbun run build\n\n# Check for errors\n# Fix and commit\n\n# Vercel should succeed on push\n```\n\n### Deployment Success but App Broken\n\n```\n1. Check Vercel logs for runtime errors\n2. Check browser console for client errors\n3. Check Supabase logs for database errors\n4. Rollback if needed\n```\n\n### Database Migrations Fail\n\n```bash\n# Check migration syntax\nsupabase migration list\nsupabase logs\n\n# For local testing:\nsupabase db reset\nsupabase db push\n```\n\n---\n\n## Emergency Contact\n\n- **Vercel Issues**: help@bachatkaro.co.in\n- **Supabase Issues**: help@bachatkaro.co.in\n- **On-Call**: [Team contact info]\n\n---\n\n*Last Updated: February 27, 2026*\n
