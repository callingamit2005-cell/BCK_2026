the Repository**\n   - Go to https://github.com/bachatkaro/repo\n   - Click \"Fork\"\n   - Clone your fork: `git clone https://github.com/YOUR_USERNAME/repo.git`\n\n2. **Add Upstream Remote**\n   ```bash\n   git remote add upstream https://github.com/bachatkaro/repo.git\n   ```\n\n3. **Create Feature Branch**\n   ```bash\n   git checkout -b feature/my-feature\n   ```\n\n---\n\n## Development Setup\n\nSee [SETUP_GUIDE.md](SETUP_GUIDE.md) for complete setup instructions.\n\n**Quick Start**:\n\n```bash\ngit clone YOUR_FORK_URL\ncd Bachatkaro\nbun install\ncp .env.local.example .env.local\n# Edit .env.local with your Supabase keys\nbun run dev\n```\n\n---\n\n## Code Style\n\n### TypeScript\n\n- Use TypeScript for all code\n- Use strict mode (`tsconfig.json`)\n- Type all function parameters and returns\n\n```tsx\n// ✅ Good\ninterface User {\n  id: string;\n  name: string;\n}\n\nfunction getUserName(user: User): string {\n  return user.name;\n}\n\n// ❌ Bad\nfunction getUser(user: any) {\n  return user.name;\n}\n```\n\n### React/TSX\n\n- Functional components only\n- Use hooks (useState, useEffect, custom hooks)\n- Props interface for all components\n\n```tsx\n// ✅ Good\ninterface ButtonProps {\n  onClick: () => void;\n  children: React.ReactNode;\n  disabled?: boolean;\n}\n\nfunction Button({ onClick, children, disabled }: ButtonProps) {\n  return (\n    <button onClick={onClick} disabled={disabled}>\n      {children}\n    </button>\n  );\n}\n\n// ❌ Bad\nfunction Button(props: any) {\n  return <button {...props} />;\n}\n```\n\n### File Naming\n\n- **Files**: `PascalCase.tsx` for components, `camelCase.ts` for utilities\n- **Directories**: `kebab-case` for feature folders\n\n```\nsrc/\n  components/\n    ExpenseCard.tsx ✅\n    expense-card.tsx ❌\n  \n  services/\n    expenseApi.ts ✅\n    expense_api.ts ❌\n  \n  utils/\n    currencyFormatter.ts ✅\n    currency-formatter.ts ❌\n```\n\n### Formatting\n\n- Use Prettier (auto-format on save)\n- Max line length: 80-100 characters\n- 2 spaces indentation\n\n```bash\n# Format code\nbun run lint -- --fix\n```\n\n### Comments\n\n```tsx\n// ✅ Explain WHY, not WHAT\nconst debounceMs = 300; // Prevent rapid re-calls during typing\n\n// ❌ Obvious comments\nconst name = 'John'; // Set name to John\n```\n\n---\n\n## Git Workflow\n\n### Branch Naming\n\n```\nfeature/add-dark-mode          ✅ New feature\nfix/crash-on-empty-list        ✅ Bug fix\ndocs/update-readme             ✅ Documentation\nrefactor/simplify-data-flow    ✅ Code cleanup\nchore/update-dependencies      ✅ Maintenance\n```\n\n### Commit Messages\n\nFollow [Conventional Commits](https://www.conventionalcommits.org/):\n\n```\nfeat: add dark mode toggle\n      Add dark mode support to Dashboard\n      \n      - Implement theme context\n      - Add toggle in settings\n      - Persist preference to localStorage\n\nfix: prevent crash on empty expense list\nrefactor: simplify expense filtering logic\ndocs: add API documentation\ntest: add tests for calculator function\nchore: upgrade React to 18.3\n```\n\n### Commit Best Practices\n\n```bash\n# ✅ Good: Small, focused commits\ngit commit -m \"feat: add expense filter by category\"\n\n# ❌ Bad: Huge commits\ngit commit -m \"update project\"\n\n# ✅ Good: Descriptive messages\ngit commit -m \"fix: prevent double submission of expense form\"\n\n# ❌ Bad: Vague messages\ngit commit -m \"fix bug\"\n```\n\n---\n\n## Writing Code\n\n### Architecture Principles\n\n1. **Single Responsibility**: Each function does one thing\n2. **Separation of Concerns**: UI, logic, and data are separate\n3. **DRY**: Don't Repeat Yourself\n4. **KISS**: Keep It Simple, Stupid\n\n### Component Structure\n\n```tsx\n// 1. Imports\nimport React from 'react';\nimport { useQuery } from '@tanstack/react-query';\n\n// 2. Types\ninterface ExpenseCardProps {\n  expense: Expense;\n  onDelete?: () => void;\n}\n\n// 3. Component\nexport function ExpenseCard({ expense, onDelete }: ExpenseCardProps) {\n  const [isDeleting, setIsDeleting] = React.useState(false);\n\n  const handleDelete = async () => {\n    setIsDeleting(true);\n    try {\n      await onDelete?.();\n    } finally {\n      setIsDeleting(false);\n    }\n  };\n\n  return (\n    <div className=\"card\">\n      <p>{expense.name}</p>\n      <p>₹{expense.amount}</p>\n      {onDelete && (\n        <button onClick={handleDelete} disabled={isDeleting}>\n          Delete\n        </button>\n      )}\n    </div>\n  );\n}\n```\n\n### Error Handling\n\n```tsx\n// ✅ Good: Handle errors explicitly\nasync function addExpense(data: ExpenseForm) {\n  try {\n    const response = await api.post('/expenses', data);\n    return response.data;\n  } catch (error) {\n    if (error.response?.status === 400) {\n      throw new Error('Invalid expense data');\n    }\n    throw error;\n  }\n}\n\n// ❌ Bad: Silent failures\nasync function addExpense(data: ExpenseForm) {\n  try {\n    return await api.post('/expenses', data);\n  } catch (error) {\n    // Silently fails\n    return null;\n  }\n}\n```\n\n### Accessibility\n\n```tsx\n// ✅ Good: Accessible form\n<label htmlFor=\"expense-name\">Expense Name</label>\n<input id=\"expense-name\" type=\"text\" />\n\n// ❌ Bad: No labels\n<input type=\"text\" placeholder=\"Name\" />\n```\n\n---\n\n## Testing\n\n### Run Tests\n\n```bash\n# All tests\nbun run test\n\n# Watch mode\nbun run test:watch\n\n# Specific test\nbun run test -- filename.test.ts\n```\n\n### Test Structure\n\n```tsx\nimport { render, screen, fireEvent } from '@testing-library/react';\nimport { ExpenseCard } from '@/components/ExpenseCard';\n\ndescribe('ExpenseCard', () => {\n  it('displays expense data', () => {\n    const expense = {\n      id: '1',\n      name: 'Chai',\n      amount: 25\n    };\n\n    render(<ExpenseCard expense={expense} />);\n    expect(screen.getByText('Chai')).toBeInTheDocument();\n    expect(screen.getByText('₹25')).toBeInTheDocument();\n  });\n\n  it('calls onDelete when delete button clicked', () => {\n    const handleDelete = jest.fn();\n    const expense = { id: '1', name: 'Chai', amount: 25 };\n\n    render(<ExpenseCard expense={expense} onDelete={handleDelete} />);\n    fireEvent.click(screen.getByText('Delete'));\n    expect(handleDelete).toHaveBeenCalled();\n  });\n});\n```\n\n### Testing Best Practices\n\n- Write tests as you write code\n- Test behavior, not implementation\n- Use meaningful test names\n- Test edge cases\n\n---\n\n## Submitting Changes\n\n### Before Pushing\n\n```bash\n# Run tests\nbun run test\n\n# Check types\nbun run build\n\n# Check code style\nbun run lint\n\n# Fix auto-fixable issues\nbun run lint -- --fix\n```\n\n### Push to GitHub\n\n```bash\n# Fetch latest from upstream\ngit fetch upstream develop\ngit rebase upstream/develop\n\n# Push to your fork\ngit push origin feature/my-feature\n```\n\n### Create Pull Request\n\n1. Go to GitHub repository\n2. Click \"New Pull Request\"\n3. Select your branch\n4. Fill in PR template:\n\n```markdown\n## Description\nFixed bug with duplicate submissions\n\n## Changes\n- Add form submission lock\n- Disable button during submission\n- Show loading state\n\n## Testing\n- [ ] Tested on desktop\n- [ ] Tested on mobile\n- [ ] No console errors\n\n## Related Issues\nCloses #123\n```\n\n---\n\n## Code Review\n\n### During Review\n\n- Be respectful and constructive\n- Address all review comments\n- Push follow-up commits\n- Don't force-push after review starts\n\n### Reviewing Others' Code\n\n```\n✅ Do:\n- Suggest improvements kindly\n- Ask questions to understand\n- Approve when satisfied\n- Point out security issues\n- Highlight potential bugs\n\n❌ Don't:\n- Be harsh or sarcastic\n- Request changes to style alone\n- Approve without reading\n- Change code without discussion\n```\n\n---\n\n## Common Issues\n\n### \"Port 5173 already in use\"\n\n```bash\n# Kill process\nlsof -i :5173\nkill -9 <PID>\n\n# Or use different port\nbun run dev -- --port 3000\n```\n\n### Tests Failing\n\n```bash\n# Clear cache\nbun run test -- --clearCache\n\n# Run single test for debugging\nbun run test -- filename.test.ts --reporter=verbose\n```\n\n### Git Conflicts\n\n```bash\n# Fetch and rebase\ngit fetch upstream develop\ngit rebase upstream/develop\n\n# Resolve conflicts in editor\n# Look for <<<<< HEAD ...\n\n# After resolving\ngit add .\ngit rebase --continue\ngit push origin feature/my-feature\n```\n\n---\n\n## Getting Help\n\n### Resources\n\n- GitHub Issues: Ask on project issues\n- Discussions: Share
