Node.js from https://nodejs.org\n# Then restart terminal\n\n# Or use NVM\nnvm install 18\nnvm use 18\nnode --version  # Should show v18.x.x\n```\n\n### \"npm/bun install hangs\"\n\n**Problem**: Installation stuck or very slow\n\n**Solutions**:\n```bash\n# Clear cache\nbun cache --rm\nnpm cache clean --force\n\n# Try with different package manager\nbun install  # if using npm\nnpm install  # if using bun\n\n# Check internet connection\nping google.com\n\n# Try with force flag\nbun install --force\n```\n\n### \"Port 5173 already in use\"\n\n**Problem**: Dev server won't start\n\n**Solutions**:\n```bash\n# Find what's using the port\nlsof -i :5173\n\n# Kill the process\nkill -9 <PID>\n\n# Or use different port\nbun run dev -- --port 3000\n```\n\n### \"Cannot find module '@/...'\"\n\n**Problem**: Path alias not resolving\n\n**Solutions**:\n```bash\n# Restart dev server (important!)\nCtrl+C\nbun run dev\n\n# Check tsconfig.json has correct paths\ncat tsconfig.json | grep paths\n\n# Should have:\n# \"@/*\": [\"./src/*\"]\n```\n\n### \"Supabase connection failed\"\n\n**Problem**: Can't connect to Supabase\n\n**Solutions**:\n```bash\n# Check .env.local exists\nls -la .env.local\n\n# Check credentials are correct\ngrep VITE_SUPABASE .env.local\n\n# Verify Supabase project is active\n# Go to https://app.supabase.com and check project status\n\n# Test connection\ncurl https://YOUR_PROJECT.supabase.co/rest/v1/\n```\n\n---\n\n## Development Issues\n\n### \"TypeScript errors in editor\"\n\n**Problem**: Red squiggles but code runs fine\n\n**Solutions**:\n```bash\n# Rebuild TypeScript\nbun run build\nnpm run build\n\n# Restart VS Code\n# Press Cmd+Shift+P → \"TypeScript: Restart TS Server\"\n\n# Check tsconfig.json syntax\ncat tsconfig.json\n```\n\n### \"Component not re-rendering\"\n\n**Problem**: State changes but UI doesn't update\n\n**Solutions**:\n```tsx\n// ❌ Wrong: Mutating state\nstate.items.push(newItem);\nsetState(state);\n\n// ✅ Correct: Create new object\nsetState([...state, newItem]);\nsetState({...state, item: newValue});\n```\n\n### \"API calls not working\"\n\n**Problem**: Requests return errors\n\n**Solutions**:\n```bash\n# Check browser network tab (F12 → Network)\n# Look for:\n# - Status 401 = Auth token invalid\n# - Status 403 = Permission denied\n# - Status 500 = Server error\n\n# For auth errors:\nsupabase.auth.getSession()  # Check token in console\n\n# Logout and login again\nawait supabase.auth.signOut()\n```\n\n### \"useState/useEffect warnings\"\n\n**Problem**: React warnings in console\n\n**Solutions**:\n```tsx\n// Warning: Missing dependency\n// ❌ Bad:\nuseEffect(() => {\n  console.log(userId);  // Missing userId in dependencies\n}, []);\n\n// ✅ Correct:\nuseEffect(() => {\n  console.log(userId);\n}, [userId]);\n```\n\n### \"Hot reload not working\"\n\n**Problem**: Changes don't appear instantly\n\n**Solutions**:\n```bash\n# Hard refresh\nCtrl+Shift+R (Windows/Linux)\nCmd+Shift+R (Mac)\n\n# Clear build cache\nrm -rf node_modules/.vite\n\n# Restart dev server\nCtrl+C\nbun run dev\n```\n\n---\n\n## Database Issues\n\n### \"Table not found\" error\n\n**Problem**: SQL or query fails on table\n\n**Solutions**:\n```bash\n# Check table exists\nsupabase db list-tables\n\n# If not exists, run migrations\nsupabase db push\n\n# Check migration files exist\nls supabase/migrations/\n```\n\n### \"Permission denied\" on database\n\n**Problem**: RLS policy blocking access\n\n**Solutions**:\n```bash\n# Verify you're logged in\nsupabase auth get-session\n\n# Check RLS policy allows your user\n# Go to Supabase Studio → SQL Editor\n# Check: SELECT * FROM expenses LIMIT 1;\n\n# Create test record\n# Verify user_id matches authenticated user\n```\n\n### \"Database migration failed\"\n\n**Problem**: Migration script has error\n\n**Solutions**:\n```bash\n# Check migration syntax locally\nsupabase db reset  # Resets local DB\nsupabase db push   # Run migrations\n\n# Check error message\nsupabase logs\n\n# Verify migration file SQL syntax\ncat supabase/migrations/20260227*.sql\n\n# For production, restore backup\n# Supabase Studio → Backups → Restore\n```\n\n### \"Slow database queries\"\n\n**Problem**: Queries taking too long\n\n**Solutions**:\n```sql\n-- Check slow queries\nSELECT query, mean_time FROM pg_stat_statements\nORDER BY mean_time DESC LIMIT 5;\n\n-- Add index if needed\nCREATE INDEX idx_expenses_user ON expenses(user_id);\n\n-- Check query plan\nEXPLAIN ANALYZE SELECT * FROM expenses WHERE user_id = $1;\n```\n\n---\n\n## Authentication Issues\n\n### \"Can't log in\"\n\n**Problem**: Login form not working\n\n**Solutions**:\n```bash\n# Verify email is confirmed in Supabase\n# Go to Supabase Studio → Authentication → Users\n\n# Check auth logs\n# Supabase Studio → Authentication → Logs\n\n# Try signing up instead\n# Verify email gets confirmation link\n\n# Check .env.local has Supabase keys\ngrep VITE_SUPABASE .env.local\n```\n\n### \"Invalid token\" error\n\n**Problem**: Authentication token expired or corrupted\n\n**Solutions**:\n```bash\n# Clear browser storage\n# DevTools → Application → Clear storage\n\n# Or in code:\nlocalStorage.clear();\nsessionStorage.clear();\n\n# Logout and login again\n```\n\n### \"Password reset not working\"\n\n**Problem**: Password reset email not sent\n\n**Solutions**:\n```bash\n# Verify email configuration in Supabase\n# Supabase Studio → Settings → Email\n\n# Check spam folder\n\n# Request new reset email\n\n# In development:\n# Go to Supabase Studio → Users → Click user\n# → \"Reset password & send email\"\n```\n\n---\n\n## API/Service Issues\n\n### \"useQuery returns undefined\"\n\n**Problem**: React Query data not loading\n\n**Solutions**:\n```tsx\n// Check loading state\nconst { data, isLoading, error } = useQuery(...);\n\nif (isLoading) return <Spinner />;\nif (error) return <Error error={error} />;\nif (!data) return <Empty />;\n\n// Verify query function works\nconst testData = await fetchExpenses();\nconsole.log(testData);  // Should have data\n\n// Check React Query DevTools\n// npm install @tanstack/react-query-devtools\n```\n\n### \"API returns 400 Bad Request\"\n\n**Problem**: Invalid request data\n\n**Solutions**:\n```bash\n# Check what you're sending\nconsole.log(requestBody);\n\n# Verify against API docs\n# Read API_DOCUMENTATION.md\n\n# Check required fields\n# All required fields must have values\n\n# Check data types\n# amount should be number, not string\n```\n\n### \"API returns 403 Permission Denied\"\n\n**Problem**: User not authorized\n\n**Solutions**:\n```bash\n# Verify you're authenticated\nconst session = await supabase.auth.getSession();\nconsole.log(session.data.session);  # Should exist\n\n# Check RLS policies allow action\n# Go to Supabase Studio → SQL Editor\n# Test query as your user\n\n# Verify user owns the resource\n# Only users can edit their own data\n```\n\n---\n\n## Deployment Issues\n\n### \"Build fails on Vercel\"\n\n**Problem**: Deployment failed\n\n**Solutions**:\n```bash\n# Build locally first\nbun run build\n\n# Fix any errors shown\n# Common: TypeScript errors, missing imports, undefined vars\n\n# Check build logs in Vercel\n# Vercel Dashboard → Project → Deployments → Failed → View logs\n\n# Ensure .env.local not in git\ncat .gitignore | grep env\n```\n\n### \"Environment variables not set\"\n\n**Problem**: App works locally but fails in production\n\n**Solutions**:\n```bash\n# Check Vercel has environment variables\n# Vercel Dashboard → Project → Settings → Environment Variables\n\n# Verify keys are correct\n# Supabase Dashboard → Settings → API → Copy keys\n\n# Redeploy after adding variables\n# Vercel Dashboard → Deployments → \"...\" → Redeploy\n```\n\n### \"Database connection fails in production\"\n\n**Problem**: Supabase connection error\n\n**Solutions**:\n```bash\n# Verify Supabase URL is correct (no typos)\n# Check HTTPS not HTTP\n\n# Verify Supabase project is active\n# Go to https://app.supabase.com\n\n# Check project region matches\n# API → Project URL should match .env\n\n# Test connection\ncurl https://YOUR_PROJECT.supabase.co/rest/v1/ -H \"Authorization: Bearer YOUR_KEY\"\n```\n\n---\n\n## Performance Issues\n\n### \"App loads slowly\"\n\n**Problem**: Slow initial page load\n\n**Solutions**:\n```bash\n# Check bundle size\nbun run build\nls -lah dist/\n\n# Should be < 500KB for main bundle\n\n# Use Chrome DevTools → Network tab\n# Look for large files\n# Check if can be lazy-loaded\n\n# Check Core Web Vitals\n# Go to: https://pagespeed.web.dev/\n```\n\n### \"List with many items slow\"\n\n**Problem**: Scrolling/rendering slow with 1000+ items\n\n**Solutions**:\n```tsx\n// Implement pagination\n<ExpenseList \n  expenses={expenses.slice(0, pageSize)}\n  hasMore={hasMore}\n  onLoadMore={loadNextPage}\n/>\n\n// Or virtual scrolling\nimport { FixedSizeList } from 'react-window';\n<FixedSizeList height={600} itemCount={1000} itemSize={35}>\n  {ExpenseRow}\n</FixedSizeList>\n```\n\n### \"React Query re-fetches constantly\"\n\n**Problem**: API called too frequently\n\n**Solutions**:\n```tsx\n// Configure stale time\nconst { data } = useQuery(\n  ['expenses'],\n  fetchExpenses,\n  {\n    staleTime: 5 * 60 * 1000,  // 5 minutes\n    cacheTime: 10 * 60 * 1000  // 10 minutes\n  }\n);\n\n// Disable automatic refetch\nrefetchOnWindowFocus: false\n```\n\n---\n\n## Still Need Help?\n\n### Check
